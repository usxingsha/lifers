Lifers Kali — 一页速查（跟着跑）
================================

【路径与账号 — 看不到脚本时先看这条】
  代码默认在 **用户 kali** 家目录：/home/kali/lifers/lifers_brain
  若终端提示符是 root@kali，则 **~ 是 /root**，~/lifers/... 常不存在 → 请：
    cd /home/kali/lifers/lifers_brain
  或先：su - kali
  若提示「找不到 show_train_progress.py」：Kali 上这份树 **不是 git clone**（打包时排除了 .git），**不要用 git pull** → 在 Windows 跑：
    lifers_brain\scripts\push_brain_and_loop_kali.ps1
  只解压合并、暂不起 tmux：**-SkipBootstrap**（默认会先 SSH pause 训练）。不要 pause：**-SkipPauseTrainFirst**。
  weights/.train_status.json 只有 **新版训练在跑/跑过** 才会生成；旧目录或训练未写入前 cat 会报「没有那个文件」。
  zsh 报 corrupt history：mv ~/.zsh_history ~/.zsh_history.bad; touch ~/.zsh_history
  若旧日志还在家目录根（~/lifers_full_stack.log 等）：bash scripts/kali_normalize_lifers_layout.sh 收拢到 ~/lifers/
  ~/claw-code 若在且为 root 所有：脚本会 rsync 到 ~/lifers/third_party/claw-code/；删原目录需：sudo rm -rf ~/claw-code  再  sudo chown -R kali:kali ~/lifers/third_party/claw-code

【布局收拢后继续训练 / 权重】（均在 kali 用户、目录 /home/kali/lifers/lifers_brain）
  bash scripts/kali_normalize_lifers_layout.sh
  bash scripts/lifers_train_ctl.sh run
  bash scripts/remote_kali_bootstrap_train_loop.sh
  # 若 tmux 里已有会话只想续跑、跳过 Markov：LIFERS_BOOTSTRAP_SKIP_MARKOV=1 bash scripts/remote_kali_bootstrap_train_loop.sh
  python3 scripts/show_train_progress.py
  tail -f ~/lifers/lifers_full_stack.log

【现在就该会的】
  tmux attach -t lifers-stack          看训练画面
  tail -f ~/lifers/lifers_full_stack.log   看循环日志（与 ~/lifers 仓库根同层）
  cd /home/kali/lifers/lifers_brain    # 推荐写绝对路径，避免 root/su 搞错 ~
  python3 scripts/show_train_progress.py   # ramp/SGD 百分比：weights/.train_status.json
  # Windows 本机一键看 Kali：lifers_brain\scripts\kali_train_status.ps1
  # VSCodium「LIFERS 总控」打开后会 SSH 轮询同一份 .train_status.json（需 lifers.kaliSshPrefix 等已配置）
  bash scripts/lifers_train_ctl.sh pause   暂停（档位边界更安全）
  bash scripts/lifers_train_ctl.sh run     继续
  bash scripts/lifers_train_ctl.sh stop    停训（需再 run 才会进 escalate）

【Windows 带代码跑】（不打断权重：tar 不含 weights/）
  cd lifers_brain\scripts
  powershell -File .\push_brain_and_loop_kali.ps1

【停旧栈 + 备份主权重 + 若有 chunk 则合并进主文件】
  bash scripts/remote_stop_old_merge_weights.sh
  注意：只用 tmux kill-session -t lifers-train，不要 kill「tmux 父进程」PID，会干掉整个 tmux server。

【多久算「正常用」】
  · UI / 桥接 / 工具链：扩展装好 + Reload → 几分钟～十几分钟。
  · 聊天像「大模型产品」：靠 stack.json → llm_ops 接外部 API；配好 key/网络 → 通常 1 小时内能日常用。
  · 只靠本地 lifers_transformer.json：小字符 LM，训再久也难达 GPT 级；可观改善常要「数天～数周」增量，仍有天花板。

【拷权重回 Windows】
  scp kali@主机:~/lifers/lifers_brain/weights/lifers_transformer.json 本机 lifers_brain\weights\
  # 或在本机仓库：lifers_brain\scripts\sync_weights_from_kali.ps1
  # 定时自动拉（mtime 变才下大文件）：sync_weights_from_kali.ps1 -Watch
  # 每 5 分钟：sync_weights_from_kali.ps1 -WatchIntervalSec 300

【训练吃满 CPU（train_sgd / escalate）】
  纯 Python 列表矩阵默认单核；安装 NumPy 后 train_sgd 自动走 BLAS 矩阵乘（多核）。
    pip install numpy   # 或 apt install python3-numpy
  禁用 NumPy 路径（调试用）：export LIFERS_USE_NUMPY=0
  Linux 拉满线程（按 CPU 核数）：export OMP_NUM_THREADS=$(nproc)
  亦可：export OPENBLAS_NUM_THREADS=$(nproc)

【自修复 stack】LIFERS_SELF_HEAL=1 默认开；缺键合并、坏 JSON 从模板恢复。
【自改代码】往 state/self_code_queue/ 放 JSON：{"rel_path":"...","new_text":"..."}；SANDBOX=0；见 playbook §9.6

【速度拉满（训练 / HTTP / 推理）】
  export LIFERS_MAX_SPEED=1
  # 可选细化：
  # LIFERS_TRAIN_SAVE_EVERY=25          # 大权重复核 JSON 每 N 步写盘（默认 MAX_SPEED 下 auto）
  # LIFERS_PAUSE_POLL_SEC=0.1           # pause 时轮询间隔
  # LIFERS_LOOP_IDLE_SEC=0              # escalate 循环空转间隔（配合 MAX_SPEED 默认已很小）
  # LIFERS_HTTP_TIMEOUT_CAP=12          # HTTP 上限秒（配合 MAX_SPEED）
  # LIFERS_LOCAL_LM_MAX_CHARS=160       # 本地小模型生成长度上限
  编辑器 settings：lifers.httpDirect=true 等价桥接里设 LIFERS_HTTP_DIRECT=1（代理端口不通时）
